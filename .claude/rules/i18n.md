# i18n — Luật chơi đa ngôn ngữ (Flutter client)

App hỗ trợ 7 ngôn ngữ: **en, vi, zh, ja, ko, es, fr**. Người dùng chọn ngôn ngữ trong
Settings → lựa chọn được lưu (`shared_preferences`) và áp dụng cho toàn bộ UI.

## Quy tắc bắt buộc

1. **KHÔNG hardcode chuỗi hiển thị** cho người dùng trong widget. Mọi text lấy từ
   `context.l10n.<key>` (extension ở `lib/core/l10n/l10n_ext.dart`).
2. **Template** là `lib/l10n/app_en.arb` — thêm key mới ở đây trước (kèm `@key`
   metadata nếu có placeholder như `{email}`, `{count}`).
3. **Mọi UI mới ⇒ thêm key vào TẤT CẢ 7 file** `lib/l10n/app_*.arb`. Thiếu key ở
   locale nào thì locale đó fallback về English (vẫn build được nhưng không nên).
4. Đặt tên key theo nhóm chức năng, camelCase: `loginTitle`, `valEmailRequired`,
   `chatDefaultTitle`, `groupName`… Tránh trùng nghĩa.
5. Sau khi sửa ARB: `flutter gen-l10n` (hoặc `flutter pub get`) sinh lại
   `lib/l10n/app_localizations*.dart`. **KHÔNG sửa tay** các file generated đó.
6. Chuỗi có biến dùng placeholder ARB (`"resendIn": "Resend in {seconds}s"`),
   KHÔNG nối chuỗi thủ công.
7. Format ngày/giờ/số theo locale qua `package:intl` (`DateFormat`,
   `NumberFormat`), không tự pad/format cứng.

## Kiến trúc

- `main.dart`: `MaterialApp.router` dùng `locale`, `AppLocalizations.supportedLocales`,
  `AppLocalizations.localizationsDelegates`.
- `lib/core/providers/locale_provider.dart`: `LocaleNotifier` (Riverpod) lưu lựa chọn;
  `null` = theo ngôn ngữ hệ thống. `kSupportedLocales` + `kLanguageNames` là nguồn
  sự thật cho danh sách ngôn ngữ + tên hiển thị native.
- Cấu hình: `l10n.yaml` (arb-dir `lib/l10n`, output-class `AppLocalizations`).

## Backend
i18n hiện ở phía client. auth-service/chat-service trả message/error tiếng Anh hoặc
mã lỗi; client tự map sang chuỗi localized. Không cần dịch backend ở giai đoạn này.
