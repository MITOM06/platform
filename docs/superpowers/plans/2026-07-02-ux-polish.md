### 1c — Mobile: Custom page transitions

**File**: `apps/client/lib/core/router/app_router.dart`

Trong `GoRouter`, thay `MaterialPage` bằng `CustomTransitionPage` với fade + slide:

```dart
// Helper tạo custom page transition
CustomTransitionPage<void> _buildPage({
  required BuildContext context,
  required GoRouterState state,
  required Widget child,
}) {
  return CustomTransitionPage<void>(
    key: state.pageKey,
    child: child,
    transitionDuration: const Duration(milliseconds: 220),
    reverseTransitionDuration: const Duration(milliseconds: 180),
    transitionsBuilder: (context, animation, secondaryAnimation, child) {
      // Fade + slight slide up (giống iOS modal)
      return FadeTransition(
        opacity: CurvedAnimation(parent: animation, curve: Curves.easeOut),
        child: SlideTransition(
          position: Tween<Offset>(
            begin: const Offset(0, 0.04),
            end: Offset.zero,
          ).animate(CurvedAnimation(parent: animation, curve: Curves.easeOut)),
          child: child,
        ),
      );
    },
  );
}
```

Áp dụng cho các route chính: `/`, `/profile`, `/settings`, `/friends`, `/notifications`.

## Issue 2 — Dialog "Lưu thay đổi" chỉ xuất hiện khi có thay đổi thật sự

### Root cause

`apps/web/app/(main)/profile/edit/page.tsx`, line 238:
```tsx
const hasUnsavedChanges = isDirty || !!avatarBlob || !!coverBlob
```

Về lý thuyết đúng, nhưng có **race condition**: `defaultValues` được set từ `user` (authStore, partial data) — sau đó `useEffect` gọi `reset(me)` khi `me` server data load xong. Nếu user nhấn Back TRƯỚC khi `me` load, `isDirty` có thể là `true` vì `defaultValues.bio = ''` nhưng chưa được reset về giá trị thật.

### Fix — `apps/web/app/(main)/profile/edit/page.tsx`

Thêm `isFormReady` flag:

```tsx
const [isFormReady, setIsFormReady] = useState(false)

useEffect(() => {
  if (me) {
    reset({
      displayName: me.displayName ?? '',
      bio: me.bio ?? '',
      dateOfBirth: me.dateOfBirth ? me.dateOfBirth.slice(0, 10) : '',
      phoneNumber: me.phoneNumber ?? '',
      gender: me.gender ?? '',
      showDateOfBirth: me.showDateOfBirth ?? !me.hideInfo,
      showPhoneNumber: me.showPhoneNumber ?? !me.hideInfo,
      showGender: me.showGender ?? !me.hideInfo,
    })
    setLocalPhone(me.phoneNumber ?? '')
    setLocalPhoneVerified(me.phoneVerified ?? false)
    setIsFormReady(true)  // ← đánh dấu form đã sẵn sàng với server data
  }
}, [me, reset])

// Chỉ tính là "có thay đổi" khi form đã load xong từ server
const hasUnsavedChanges = isFormReady && (isDirty || !!avatarBlob || !!coverBlob)
```

`handleBack` đã check `hasUnsavedChanges` rồi — không cần thay đổi gì thêm.

## Issue 4 — Chữ "Profile" trên góc bị che/cắt trong UserProfileDrawer

### Root cause

Trong `UserProfileDrawer.tsx`:
```tsx
<ResponsiveModal
  title={t('profileTitle')}   // → "Profile"
  className="max-w-sm overflow-hidden p-0 gap-0"
>
  <div className="mt-4 space-y-4 px-6 pb-6">
    <div className="relative -mx-6 -mt-4 h-24 ...">  {/* cover photo */}
```

`ResponsiveModal` renders `DialogTitle` ("Profile") bên trên `children`. Với `gap-0` và `p-0` trên `DialogContent`, cover photo có `-mt-4` kéo ngược lên ĐÈ LÊN chữ "Profile" → chữ bị che một phần.

### Fix — `UserProfileDrawer.tsx`

Không pass `title` prop vào `ResponsiveModal` (chữ "Profile" không cần thiết secara visual — modal context đã rõ). Thay bằng hidden accessible title bên trong content:

```tsx
<ResponsiveModal
  open={userId !== null}
  onOpenChange={(o) => { if (!o) onClose() }}
  // KHÔNG pass title ở đây nữa — tránh DialogTitle bị cover photo đè lên
  className="max-w-sm overflow-hidden p-0 gap-0"
>
  {isLoading ? (
    <div className="flex justify-center py-12">
      <Loader2 className="size-6 animate-spin text-muted-foreground" />
    </div>
  ) : (
    <div className="mt-4 space-y-4 px-6 pb-6">
      {/* Screen-reader only title — accessible nhưng không hiện trên UI */}
      <span className="sr-only">{t('profileTitle')}</span>

      {/* Cover photo ... */}
```

`ResponsiveModal` khi không có `title` prop sẽ không render `DialogHeader` → không có text nào bị đè. Accessibility vẫn đảm bảo qua `sr-only` span bên trong.

Kiểm tra `responsive-modal.tsx`: khi `title` là `undefined`, `DialogHeader` không render → không cần thay đổi `ResponsiveModal` component.

## Issue 5 — Cải thiện chất lượng hình ảnh / video

### Phân tích hiện trạng

- **Upload**: `chatService.uploadFile(blob, filename)` — blob từ `react-easy-crop` (avatars/covers) hoặc file picker (chat images)
- **Serve**: `absoluteMediaUrl(url)` → URL tới chat-service endpoint
- **Download**: `downloadMediaUrl(url)` → thêm `?download=true`
- **Không có compression param** hiện tại — server phục vụ file nguyên bản

Vấn đề thực tế:
1. `react-easy-crop` export canvas với quality mặc định (0.92) → avatar/cover bị giảm chất lượng
2. Next.js `<Image>` với `unoptimized` prop bỏ qua Next.js optimization (đúng vì ảnh từ external URL)
3. Không có HD mode cho image viewer
4. Video player không có quality selector

### 5a — Tăng quality khi crop avatar/cover

**File**: `apps/web/components/profile/ImageCropperModal.tsx`

Tìm chỗ `canvas.toBlob(...)` — thay quality từ mặc định lên `0.95`:

```tsx
// Trước (mặc định hoặc lower quality):
canvas.toBlob((blob) => { ... }, 'image/jpeg')

// Sau:
canvas.toBlob((blob) => { ... }, 'image/webp', 0.95)
// Dùng webp nếu browser support (nhỏ hơn jpeg cùng quality), fallback jpeg:
const mimeType = canvas.toDataURL('image/webp').startsWith('data:image/webp') ? 'image/webp' : 'image/jpeg'
canvas.toBlob((blob) => { if (blob) onCrop(blob) }, mimeType, 0.95)
```

### 5c — Download luôn phục vụ file gốc

`downloadMediaUrl` hiện tại thêm `?download=true`. Đảm bảo chat-service backend khi nhận `download=true` phục vụ file với `Content-Disposition: attachment` và **không compress**. Đây là backend concern — nếu backend đang resize/compress khi serve, cần thêm param:

```tsx
// Nếu backend support quality param:
export function downloadMediaUrl(url: string, hd = true): string {
  const base = absoluteMediaUrl(url)
  const sep = base.includes('?') ? '&' : '?'
  return `${base}${sep}download=true${hd ? '&quality=original' : ''}`
}
```

Nếu backend KHÔNG compress hiện tại → không cần thay đổi, keep as is.

### 5e — Video: quality note

Video player hiện tại phục vụ video nguyên bản (không transcoding). Không cần thêm quality selector nếu server đã serve file gốc. Chỉ cần đảm bảo:
- Web: `<video>` không compress khi load (đây là browser behavior, không phải code)
- Mobile: `VideoPlayerController.networkUrl` load URL gốc — không cần thay đổi

**Nếu sau này muốn transcoding** (480p/720p/1080p), cần backend FFmpeg pipeline — đây là task riêng, không thuộc scope plan này.
