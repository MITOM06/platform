## Notes

- `errGeneric` does not exist in the ARB — the fallback in `authErrorToString` uses the pre-existing `errActionFailed` key ("Something went wrong. Please try again.") which is semantically identical.
- Login screen error path goes through `AuthNotifier` → `AsyncValue.guard` → listener; the helper is invoked in `_friendlyError` which already receives the raw `Object error` — a `DioException` type-check is added there so the helper is called correctly.
- All backward-compatible existing keys (`errInvalidCredentials`, `errEmailExists`, etc.) are preserved.
